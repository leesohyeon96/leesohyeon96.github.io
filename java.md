<br/>

## JVM이란?

- **가상 기계 => 현실 세계(프로그램[소프트웨어]은 개발자가 개발 도구를 이용해 개발하고 운영체제를 통해 물리적 컴퓨터인 하드웨어 상에서 구동)를 그대로 모방**
- **자바가 만들어주는 가상 세계도 위의 설명과 마찬가지**
- **`JDK(Java Development Kit, 자바개발도구)`를 이용해 개발된 프로그램은 `JRE(Java Runtime Environment, 자바실행환경)`에 의해 `가상의 컴퓨터인 JVM(Java Virtual Machine, 자바가상기계)` 상에서 구동됨**

- **JVM이 맨 먼저 하는 일은 `전처리`라고 하는 과정이고 가장 먼저 java.lang 패키지를 static영역에 가져다 놓음**
- **이로 인해, System.out.println() 같은 메소드를 사용할 수 있게 되는 것**
- **(예를 들어 main()메소드라는 것이 있다고 하면) main()메소드가 끝나면 JRE는 JVM을 종료하고 본인 자체도 `운영체제 상의 메모리에서 사라지게 됨` **
<br/>

## 클래스

- **객체를 위한 factory같은 것**

- **같은 속성 가지고 있지만, 데이터는 다름!**
<br/>

## 클래스와 객체의 차이

- **클래스 : 객체 = 펭귄 : 뽀로로**
- **펭귄의 나이는? -> 알수없음 **
- **뽀로로의 나이는? -> 검색해보면 나옴**
- **즉, 나이를 알 수 있는 뽀로로는 '객체'  / 나이를 알 수 없는 펭귄은 '클래스' **
- **클래스 -> `분류의 개념`이며 `분류/집합/같은 속성,기능을 가진 객체를 총칭하는 개념`**
<br/>
## *그럼 인스턴스는?

- **클래스를 이용해 Object를 만든 것을 강조할 때는 object라는 표현보다는 `클래스의 인스턴스(instance)`라는 표현을 씀**
- **이때, `클래스를 객체의 설계도`라고 말할 수 있음 -> 붕어빵과 붕어빵틀의 비유는 객체의 이런 일면을 설명하기 위한 것임**
